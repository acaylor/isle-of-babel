# CLAUDE.md

Isle of Babel is a quiet, Myst-like exploration game in a wizard's pocket
dimension, built with Godot 4.7. Read `docs/ARCHITECTURE.md` before
structural changes, `docs/VISION.md` before designing features, and
`docs/CONTRIBUTING.md` for recipes. This file is the operational summary.

## Hard rules (violating these breaks the project's premise)

1. **The repository is 100% text. Never commit a binary.** Every mesh,
   material, and page of prose is generated by GDScript at runtime.
   `.gitattributes` pre-routes binaries through LFS, but adding any is a
   deliberate decision made with AJ, not in a PR.
2. **All randomness is seeded.** Every generator makes its own
   `RandomNumberGenerator` with a fixed constant seed. The world must be
   byte-identical on every launch. Library cells derive seeds from grid
   coordinates.
3. **`height_at(x, z)` is the single source of terrain truth** per outdoor
   scene. Never hardcode a height; add terrain features as terms inside it.
   Anything terrain must meet flush (bridges, abutments) is planned in
   `_ready()` *before* the terrain mesh builds ("plan before terrain").
4. **Vertex colors are sRGB** — use `Forge.vc_mat()`, and `CULL_DISABLED` +
   hand-computed normals for triangle-soup meshes (see `Flora._finish`).
   Skipping either yields washed-out colors or invisible faces.
5. **The tone is quiet, literate, gently melancholy.** No combat, timers,
   dialogue trees, or fail states beyond the lake politely returning you to
   the dock. In-game prose gets the same review rigor as code — read
   `scripts/book_lore.gd` before writing any lore; written texts carry the
   story, generated ones carry atmosphere. `docs/VISION.md` lists non-goals.

## Commands

The Godot binary on this machine (brew cask, not on PATH):
`/Applications/Godot.app/Contents/MacOS/Godot`

```sh
# fresh checkout only — without this, every class_name fails to resolve
godot --headless --path . --import

godot --path .                                   # run the game
godot --headless --path . res://tests/smoke.tscn # full smoke test → "SMOKE OK"

# render frames to look at (visual changes are verified by looking):
CAP_SCENE=res://scenes/forest.tscn CAP_CAM="-44,18,-68" CAP_LOOK="-58,11,-84" \
  godot --path . res://tests/capture.tscn \
  --write-movie /tmp/shots/f.png --fixed-fps 10 --quit-after 8
# CAP_SPAWN=jetty|voyage|dock|menu... for the player's view (omit CAP_CAM)
```

## Testing discipline

- The smoke test (`tests/observer.gd`, time-based stages) must pass before
  every commit. New mechanics get a stage; bug fixes get an assertion.
- Simulate real input, not method calls, where the bug lived in input:
  `Input.parse_input_event` for keys, `Input.action_press` for held walks.
  The suite literally walks the footbridge, the tower ramp, and into the
  boundary cliffs.
- **Prove the test can fail**: temporarily reintroduce the bug, run, quote
  the failure in the commit/PR, restore. Back up the working file first —
  `git checkout --` once ate an uncommitted fix along with the experiment.
- Verify visuals by rendering captures and reading the images; include
  screenshots in PRs for anything visible.

## Workflow

- Feature branch → PR on Gitea (`aj/isle-of-babel` at git.lan.ayjc.net, via
  the gitea MCP tools) → **AJ merges → merge is the release point** → stamp
  the CHANGELOG date on `main`, push, tag `vX.Y.Z-alpha` as a pre-release.
- Bump `Game.VERSION` (autoload/game.gd) in the PR that starts a version;
  changelog entries go under an `Unreleased`/versioned heading as you work.
- Gitea is canonical. GitHub (`acaylor/isle-of-babel`) is an auto-mirror
  (Gitea push mirror, syncs on commit) — never push feature work directly
  to the `github` remote; it follows by itself. Exception: the orphan
  `gh-pages` branch (the project site) is pushed to **both** remotes.
- GitHub releases are recreated manually via `gh release create` after
  tagging (the mirror moves refs, not release objects).
- Playtest feedback from AJ is the primary QA; his reported bugs become
  named regression tests.

## Code style

- Tabs; fully typed GDScript (`var x := ...`, typed params/returns).
  Iterating a literal array needs a typed loop var (`for side: float in
  [-1.0, 1.0]`) or type inference errors follow.
- Comments state constraints the code can't show, not narration.
- Interactables are `Interactable.make(shape, prompt, callable, xform)` —
  `prompt`/`action` are mutable at runtime (that's how the cold ring wakes).
- Cheap ambient animation goes through the scene's FX registries (pulse
  materials/lights, orbit pivots, drift motes) in one `_process` pass.

## Known traps

- Input events process before physics in the same frame: an action that
  closes UI can immediately re-trigger via `is_action_just_pressed` in
  `_physics_process` — consume the press (see `_consume_interact`).
- Player walks ≤55° slopes (`floor_max_angle`); walkable routes should stay
  under ~45°, barriers over ~60° — and get a walk-test.
- The 1600 m water plane underlies everything at y=0; terrain below that
  shows lake. `fall_reset_y` rescues the player per scene.
- UI: never position a Control before its text/size exists — use containers
  or `grow_horizontal = GROW_DIRECTION_BOTH` (bit us twice).
